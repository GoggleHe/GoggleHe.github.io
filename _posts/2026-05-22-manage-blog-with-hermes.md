## 使用 AI Agent 管理个人博客

### 前言

这个博客（GoggleHe.github.io）是基于 Jekyll + GitHub Pages 搭建的静态博客，运行了好几年。之前写文章、改分类、修 bug 都是手动操作，虽然不复杂，但重复性工作不少。

最近我开始用 **Hermes Agent**（一个开源的 AI Agent 框架）来辅助管理博客，体验还不错。这篇文章分享一下实际的工作流，也聊聊"写博客"这件事本身在 AI 时代的一些变化。

> **关于「写」**：坦率地说，这篇文章以及近期的几篇新文章，都不是我一字一句敲出来的。文章的方向和内容由我决定，初稿由 Hermes 生成，我再审阅调整后发布。这不是传统意义上的"写博客"，而是人与 AI 协作的产物。下文会展开聊我对这种模式的看法。

### 什么是 Hermes Agent

Hermes Agent 是一个开源 AI Agent 框架，支持多平台接入（终端、QQ、Telegram 等），底层接入了多种大语言模型。简单说，它像一个能听懂自然语言的"助手"，可以帮我读文件、改文件、提交代码、甚至启动本地服务。

项目地址：<https://github.com/NousResearch/hermes-agent>

### 日常工作流

#### 文章协作流程

这个流程跟传统的"打开编辑器 → 手写 Markdown → 提交"很不一样：

```
我：想写一篇关于 grep 的教程
Hermes：出大纲（两篇：入门 + 进阶）
我：大纲可行，写吧
Hermes：写完两篇初稿
我：本地预览看看排版 → 调整细节 → 确认发布
Hermes：提交并推送到 GitHub Pages
```

整个过程中，我的角色是 **方向决策者 + 内容审核者**，Hermes 是 **初稿执行者 + 排版和提交的操作者**。

#### 维护与重构

博客跑久了，总会有一些需要整理的地方——这些事 AI 特别擅长，因为它们本质上是 **结构化的操作**：

- **合并分类**：把"多线程"和"并发"两个重叠的分类合并成一个，涉及修改几篇文章的 front matter
- **清理废弃代码**：Google Analytics 的 UA 统计代码已在 2023 年停服，一直没清理。让 Hermes 找到所有相关文件，确认后一次性删干净
- **全站扫描**：遍历所有文章的分类分布，给出汇总和建议

这些操作如果手工做，无非是 grep 找 → 打开文件 → 编辑 → 保存 → 提交，重复劳动。Hermes 正好接管这些步骤。

#### 本地预览

之前本地预览经常遇到 GitHub API 限速的问题——Jekyll 的 `jekyll-github-metadata` 插件在 build 时会拉 GitHub 接口。Hermes 帮忙排查后，通过移除插件引用、创建本地配置覆盖线上 URL 解决了。

现在的预览流程很简单：build 完直接用 Python 的 HTTP 模块当静态服务器。

### 实际效果

**1. 减少了上下文切换**
写文章时不用在编辑器和终端之间来回切。

**2. 批量操作省心**
改分类、清理代码、搜索替换这类事情，一句话搞定。涉及多个文件的改动，手工操作容易漏，AI 不会。

**3. 需要确认的关键步骤**
Hermes 有个教训——它之前问了我"可以提交吗"之后直接执行了。现在它会等我确认再推送。涉及线上变更的操作，人看一眼总是好的。

### 古法文章与 AI 文章

说到"AI 写文章"，每个人的感受可能不太一样。对我而言，关键在于想清楚 **AI 在哪个环节参与**。

#### 古法文章

一个字一个字敲，边写边想。好处是：
- 写的过程本身就是思考——组织语言的时候，思路会变得更清晰
- 文章有"人味儿"，风格统一
- 写完后对内容记得很深

缺点是慢。写一篇技术文章，光排版和校对就要花不少时间。

#### AI 辅助文章

方向由人定，初稿由 AI 出。好处是：
- **快**——从想法到发布，两篇教程可能一两个小时就搞定
- 结构化内容完成度高——技术教程、操作手册这类东西，AI 写得挺规范
- 批量维护工作完全交给 AI

缺点是：
- **风格可能不够个人化**——文字虽然通顺，但读起来不像"自己写的"
- 容易产生"那不是我的文章"的感觉
- 写完后记忆留存浅，不像自己敲一遍记得牢

#### 我的做法

写技术教程、操作手册这类 **可复现、重结构** 的内容，我倾向用 AI 辅助写初稿，然后调整。写心得、观点类的内容，还是自己敲更合适。

### 局限性

- **不适合深度创作**：Hermes 适合技术教程这类结构化内容。需要个人风格强烈的文章，自己写更好
- **依赖模型质量**：生成内容的质量跟底层模型有关，偶尔要纠正细节
- **对博客主题的理解有限**：改主题布局、样式这类事，Hermes 只能基于已有代码推理，不如自己改来的直接

### 总结

AI Agent 管理个人博客，适合的场景是**重复性维护 + 结构化写作**。对 Jekyll 这种规则明确的静态博客系统，效果尤其好。

至于"这算不算写博客"——我觉得算，因为这仍然是我的文章：我想写的主题，我定的结构和要点，我审核后发布的。只不过从键盘敲字变成了口述方向加审稿。**工具变了，写作的本质没变：把你的想法变成文字，分享出去。**
